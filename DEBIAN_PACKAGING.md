# Debian Packaging for rpi-fastbootd

## Overview

This package uses Debian devscripts for native packaging, replacing the previous CPack-based system while maintaining all features including automatic version calculation from git.

## Quick Start

```bash
# Build package
./build-package.sh

# Build without libcryptsetup (Apache 2.0 license)
./build-package.sh --no-cryptsetup

# Build signed package
./build-package.sh --signed
```

## Version Calculation

The package version is automatically calculated from git:

```
Format: 14.0.0-<git-revision>
Example: rpi-fastbootd_14.0.0-247_arm64.deb
```

Where:
- `14.0.0` = Android 14 baseline version (from CMakeLists.txt)
- `<git-revision>` = `git rev-list --count 2bdf18c51^..HEAD`

This matches the previous CPack behavior exactly.

### How It Works

1. **debian/gen-version.sh** - Script that calculates version and generates debian/changelog
2. **debian/rules** - Calls gen-version.sh before building
3. **build-package.sh** - Also calls gen-version.sh for convenience

The calculation happens at build time, so the version is always up-to-date.

## Network Access for FetchContent

The build system uses CMake FetchContent to build jsoncpp statically in-tree for consistent, reproducible builds:

- `debian/rules` exports `DEB_BUILD_OPTIONS += network` to allow CMake to download jsoncpp (v1.9.6)
- Static linking eliminates runtime dependency on system libjsoncpp
- For offline builds, you can use `-DUSE_SYSTEM_JSONCPP=ON` and install `libjsoncpp-dev`

## Building

### Using build-package.sh (Recommended)

```bash
# Standard build with libcryptsetup (GPLv3)
./build-package.sh

# Without libcryptsetup (Apache 2.0)
./build-package.sh --no-cryptsetup

# Signed package
./build-package.sh --signed
```

### Manual Build

```bash
# Generate version
./debian/gen-version.sh

# Build package
dpkg-buildpackage -b -uc -us -j$(nproc)

# Or with debuild
debuild -b -uc -us

# With specific options
DEB_BUILD_OPTIONS="nocryptsetup" dpkg-buildpackage -b -uc -us
```

### Build Options

Set `DEB_BUILD_OPTIONS` environment variable:

- `nocryptsetup` - Build without libcryptsetup (Apache 2.0 license)
- `nocheck` - Skip running tests
- `nostrip` - Don't strip debug symbols

Example:
```bash
DEB_BUILD_OPTIONS="nocryptsetup nostrip" ./build-package.sh
```

## Installation

```bash
# Install package
sudo dpkg -i ../rpi-fastbootd_14.0.0-<revision>_arm64.deb

# Install dependencies if missing
sudo apt-get install -f

# Check installation
dpkg -l | grep rpi-fastbootd
systemctl status fastbootd.service
```

## Comparison with CPack

| Feature | CPack | Debian devscripts |
|---------|-------|-------------------|
| Version calculation | ✅ git-based | ✅ git-based (same algorithm) |
| Dependencies | ✅ Specified | ✅ Specified + auto-detected |
| Systemd integration | ✅ Manual install | ✅ dh_installsystemd |
| Documentation | ✅ Included | ✅ Included + Debian-specific |
| Build artifacts | dist/ directory | Parent directory |
| Package naming | rpi-fastbootd_VERSION_ARCH | Same format |
| Licensing | ✅ Apache 2.0/GPLv3 | ✅ Apache 2.0/GPLv3 |
| nocryptsetup option | CMake flag | DEB_BUILD_OPTIONS |
| Debug symbols | Stripped | dbgsym package |
| Standards compliance | Basic | Debian Policy compliant |
| Repository integration | Manual | apt-get compatible |

## Advantages over CPack

1. **Standards Compliant** - Follows Debian Policy
2. **Better Dependency Handling** - Auto-detection via dh_shlibdeps
3. **Debug Symbols** - Automatic dbgsym package creation
4. **apt Integration** - Works with apt-get, apt-cache, etc.
5. **Repository Ready** - Can be added to apt repository
6. **Lintian Checks** - Automatic quality checks
7. **Systemd Integration** - Proper dh_installsystemd handling
8. **Maintainer Scripts** - postinst/prerm if needed
9. **Build Profiles** - Standard DEB_BUILD_OPTIONS support
10. **Git Integration** - git-buildpackage workflow

## Updating the Package

When making changes:

1. Version is auto-calculated from git
2. Update debian/changelog if needed for release notes
3. Rebuild: `./build-package.sh`

For release versions:
```bash
dch -v 14.0.0-<revision> "Release notes"
./build-package.sh --signed
```

## Troubleshooting

### Version shows 14.0.0-0

Run: `./debian/gen-version.sh` to regenerate changelog

### Build fails with "changelog not found"

Ensure debian/changelog exists (placeholder is in git)

### Wrong dependencies listed

Check: `dpkg-deb -f ../rpi-fastbootd_*.deb Depends`
Dependencies are auto-detected by dh_shlibdeps

### Package won't install

Check dependencies: `sudo apt-get install -f`
Or manually install missing packages
